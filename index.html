flexdashboard: Easy interactive dashboards for R

  • Use R Markdown to publish a group of related data visualizations as a dashboard.

  • Ideal for publishing interactive JavaScript visualizations based on htmlwidgets (but also works with standard R graphics).

  • Flexible and easy to specify layouts, including automatic re-sizing of charts to fill available browser space.

  • Optionally use Shiny to drive visualizations dynamically.

Getting Started

Install the flexdashboard package from GitHub as follows:

devtools::install_github("rstudio/flexdashboard")

To author a flexdashboard you create an R Markdown document with the flexdashboard::flex_dashboard output format. You can do this from within RStudio using the New R Markdown dialog:

If you are not using RStudio you can create a new flexdashboard R Markdown file from the command line as follows:

rmarkdown::draft("dashboard.Rmd", template = "flex_dashboard", package = "flexdashboard")

The basic mechanics of creating and laying out a dashboard are described below. You can create your own custom layout from scratch or copy and modify one of the pre-canned ones from the Layouts page. You can also optionally use Shiny to make your dashboards more dynamic and interactive.

Dashboard Layout

You can use flexdashboard to publish groups of related data visualizations as a dashboard. One of the important characteristics of these dashboards is that charts within them are automatically re-sized to fit available browser width and (optionally) browser height. This makes them ideal for publishing htmlwidgets since these widgets automatically re-size themselves to their containers.

You can also use flexdashboard to publish base, lattice, and ggplot2 graphics. These charts are also re-sized automatically however a bit more work (described below in Chart Sizing) is required to ensure that they are properly scaled. Even with this extra work standard R graphics won’t always fit perfectly into their viewport, so in general htmlwidget based charts will yield better results. Note that the plotly package enables rendering of ggplot2 graphics as interactive htmlwidgets.

Chart Sizing

Note that by default the charts flow down the page and the page scrolls if they occupy more space than is available. However, you might prefer that all charts be automatically sized to fill the containing web page. To do that you specify the fill_page option (see below for an example)

By default charts split space equally with other charts in their row or column. You may however wish to emphasize certain charts by allocating more space to them. The space allocated to a chart is determined by the knitr fig.width and fig.height options, which by default use a width of 5 inches and a height of 3.5 inches. If we wanted to allocate more vertical space to the single chart in the first row we could change it’s fig.width and fig.height accordingly.

Figure Sizes and Scaling

Charts will always be automatically scaled to fit within their allocated region. For charts based on htmlwidgets (JavaScript) this scaling will nearly always result in a chart that perfectly fits the bounds of it’s container. However, for charts based on standard R graphics the chart will be scaled so that it’s original aspect ratio is preserved. Therefore, the closer the specified fig.width and fig.height are to the actual laid out size of the chart the better it will scale.

The Multiple Pages section below includes some examples of specifying explicit fig.width and fig.height values that are well tailored to the containers they will be laid out within.

Visual Themes

A variety of themes are available to modify the appearance of flexdashboard. Supported themes include “default”, “bootstrap”, “cerulean”, “journal”, “flatly”, “readable”, “spacelab”, “united”, “cosmo”, “lumen”, “paper”, “sandstone”, “simplex”, and “yeti”. Note that the “cosmo” theme is used when “default” is specified.

These themes are all based on the themes available in the R Markdown package (which are in turn based on Bootswatch themes).
