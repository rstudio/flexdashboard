flexdashboard: Easy interactive dashboards for R

  • Use R Markdown to publish a group of related data visualizations as a dashboard.

  • Ideal for publishing interative JavaScript visualizations based on htmlwidgets (but also works with standard R graphics).

  • Flexible and easy to specify layouts, including automatic resizing of charts to fill available browser space.

  • Optionally use Shiny to drive visualizations dynamically.

Examples

Getting Started

Install the flexdashboard package from GitHub as follows:

devtools::install_github("rstudio/flexdashboard")

To author a flexdashboard you create an R Markdown document with the flexdashbaord::flex_dashboard output format. The basic mechanics of creating and laying out a dashboard are described below. You can create your own custom layout from scratch or copy and modify one of the pre-canned ones from the Layouts page.

Once you’ve mastered the basics check out the Advanced page for documentation on themes, multi-page layouts, and including short text commentary within your dashboards. You can also optionally use Shiny to make your dashboards more dynamic and interactive.

Dashboard Layout

You can use flexdashboard to publish groups of related data visualizations as a dashboard. One of the important characteristics of these dashboards is that charts within them are automatically resized to fit available browser width and (optionally) browser height. This makes them ideal for publishing htmlwidgets since these widgets automatically resize themselves to their containers.

Note that you can also use flexdashboard to publish base, lattice, and ggplot2 graphics. These charts are also resized automatically however a bit more work (described below in Chart Sizing) is required to ensure that they are properly scaled.

In this case rows are defined using level 2 markdown headers (------------------) and charts are defined using level 3 headers (###). Charts are laid out horizontally within a row. Chart Sizing

Note that by default the charts flow down the page and the page scrolls if they occupy more space than is available. However, you might prefer that all charts be automatically sized to fill the containing web page. To do that you specify the fill_page option (see below for an example)

By default charts split space equally with other charts in their row or column. You may however wish to emphasize certain charts by allocating more space to them. The space allocated to a chart is determined by the knitr fig.width and fig.height options, which by default use a width of 5 inches and a height of 3.5 inches. If we wanted to allocate more vertical space to the single chart in the first row we could change it’s fig.width and fig.height accordingly.

Figure Sizes and Scaling

Charts will always be automatically scaled to fit within their allocated region. For charts based on htmlwidgets (JavaScript) this scaling will nearly always result in a chart that perfectly fits the bounds of it’s container. However, for charts based on standard R graphics the chart will be scaled so that it’s original aspect ratio is preserved. Therefore, the closer the specified fig.width and fig.height are to the actual laid out size of the chart the better it will scale.
